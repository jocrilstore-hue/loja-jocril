
# Prompt Pathologies

Prompt pathologies are **failure modes that emerge over time** when prompts are iterated, optimized, or generated by other prompts.

Unlike prompt anti-patterns (which affect a single prompt), pathologies appear when prompts evolve across multiple iterations, agents, or model interactions.

This document describes the most common prompt pathologies and how to detect and correct them.

---

# 1. Fractal Prompt

## Definition

A **Fractal Prompt** occurs when a prompt asks the model to generate instructions for another AI instead of performing the task directly.

Structure:

Model → Prompt → Model → Prompt → Task

Instead of the simpler and more effective:

Model → Task

---

## Example

Bad prompt:

Write a prompt for another AI that will analyze the dataset and provide insights.

What happens:

The model generates meta-instructions rather than solving the problem.

---

## Why It Happens

Fractal prompts appear when:

- prompt engineering tools recursively generate prompts
- AI writes prompts for other AI systems
- prompt optimization is prioritized over task execution
- users reuse AI-generated prompts without simplifying them

---

## Problems Caused

Fractal prompts cause:

- unnecessary abstraction layers
- larger prompts without better results
- model confusion about execution vs instruction design
- increased token usage

---

## Correction

Convert meta-instructions into direct task instructions.

Bad:

Write a prompt that tells an AI how to analyze the dataset.

Better:

Analyze the dataset and produce:

- key trends
- anomalies
- recommendations

Return the results as structured JSON.

---

# 2. Prompt Drift

## Definition

Prompt Drift occurs when a prompt gradually changes across iterations until it no longer performs the original task.

Small changes accumulate over time and alter the behavior of the prompt.

---

## Example

Original prompt:

Summarize the document in five bullet points.

After multiple revisions:

Explain the main ideas in the document.

The task has shifted from **structured summarization** to **open-ended explanation**.

---

## Why It Happens

Prompt drift often occurs when:

- prompts are rewritten repeatedly
- AI rewrites prompts for optimization
- instructions become generalized
- original constraints disappear

---

## Symptoms

Signs of drift include:

- output format changes
- responses become longer or more conversational
- the task interpretation changes
- different runs produce inconsistent outputs

---

## Prevention

Use prompt anchors.

Example:

TASK (DO NOT MODIFY):

Summarize the document in five bullet points.

Structured output schemas also reduce drift.

---

# 3. Prompt Collapse

## Definition

Prompt Collapse occurs when prompts become overly complex or over-optimized, causing them to perform worse.

The prompt becomes brittle and sensitive to small input variations.

---

## Example

Over-engineered prompt:

You are a highly specialized expert summarization AI trained in academic discourse analysis. Carefully analyze the document using a structured reasoning framework and produce a refined executive summary following professional editorial standards.

Despite sounding sophisticated, this often performs worse than:

Summarize the document in five bullet points.

---

## Why It Happens

Collapse occurs when prompts accumulate:

- excessive role descriptions
- multiple layers of instructions
- redundant constraints
- stylistic directives

The model spends more effort interpreting the prompt than solving the task.

---

## Symptoms

Collapsed prompts may show:

- slower responses
- inconsistent behavior
- ignored instructions
- hallucinated formatting

---

## Prevention

Favor simplicity.

When improving prompts, attempt to remove instructions before adding new ones.

---

# 4. Prompt Illusion

## Definition

Prompt Illusion occurs when a prompt change appears to improve results but the improvement is actually due to randomness or sampling variation.

The prompt change is incorrectly credited for the improvement.

---

## Example

Prompt change:

Analyze the dataset carefully.

to

Carefully analyze the dataset and think deeply.

A better output occurs once, leading the engineer to believe the wording improved the prompt.

In reality, the change had no causal effect.

---

## Why It Happens

LLM outputs are probabilistic.

Two runs with the same prompt may produce different results.

Without controlled testing, improvements may be misattributed to prompt wording.

---

## Symptoms

Prompt illusion often leads to:

- superstition-like prompt phrases
- unnecessary wording changes
- increasingly complex prompts
- difficulty reproducing results

---

## Prevention

Use controlled evaluation.

Test prompts with:

- identical inputs
- multiple runs
- consistent evaluation criteria

If improvement persists across runs, the change is real.

If not, it is likely an illusion.

---

# Summary

Prompt pathologies appear when prompt systems evolve over time.

The four most common are:

Fractal Prompt — prompts generating prompts instead of solving tasks  
Prompt Drift — prompts gradually changing behavior  
Prompt Collapse — prompts becoming overly complex and brittle  
Prompt Illusion — false improvements caused by randomness

Understanding these pathologies helps maintain **clear, stable, and effective prompts**.
