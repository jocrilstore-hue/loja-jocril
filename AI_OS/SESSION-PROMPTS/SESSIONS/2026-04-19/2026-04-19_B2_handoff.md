## Gotchas discovered

1. **Clerk peer-dep conflict with React 19.0.0.** `@clerk/nextjs@6.39.2` declares `react@"~19.0.3 || ~19.1.4 || ~19.2.3 || ~19.3.0-0"` — project pins `19.0.0` exactly. Installed with `--legacy-peer-deps`. No runtime impact, but future `npm install` will hit it. Fix options when convenient: bump React to `^19.0.3` or stay on `--legacy-peer-deps`.
2. **`middleware.ts` deprecated in Next 16.** Use `proxy.ts`. Old project already did; we matched.
3. **`useSearchParams()` needs `<Suspense>`.** Entrar page body now sits inside a `<Suspense fallback={null}>` so static generation doesn't bail.
4. **Custom admin-login 2FA UI sacrificed.** The design bundle shipped a separate `/admin/login` page with email + password + 6-digit 2FA code input. Wiring Clerk 2FA headless was out of B2 scope — replaced with a simple redirect. Original UI file is in git history (commit before B2) if we want to revive it in B6/B7 when admin ops land.
5. **Clerk's `redirect_url` is absolute.** When Clerk's `auth.protect()` redirects unauth users, the `redirect_url` query param is the full URL, not a path. `router.push(absoluteUrl)` works in Next 16 but same-origin validation would be prudent later.
6. **Zombie dev servers on port 3000.** Same issue Maria flagged in the B1 handoff. If `npm run dev` or `start` complains, kill PID (this session: 36188 killed after verification).
