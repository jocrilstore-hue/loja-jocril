## What each page shows

### /conta
- **Real**: first name from `user.fullName ?? user.firstName`, email from `user.primaryEmailAddress`, member-since from `user.createdAt` formatted pt-PT
- **Still mock**: order count (12), tier progress (€ 852 / € 1000), address book, order rows in overview/orders tabs
- **TODO B5b**: wire order count and tier from real DB; wire address book

### /encomendas
- **Real**: all orders for the authenticated customer from Supabase; order_number, date (pt-PT), total_amount_with_vat, status badge, payment_status badge
- Each row links to `/encomenda/[order_number]`
- Empty state: "Ainda não tem encomendas." + link to /produtos
- Unauthenticated: shows empty state (no orders found)

### /encomenda/[id] and /conta/encomenda/[id]
- **Real**: order_number, status, payment_status, created_at, items (product_name, size_format, sku, qty, unit_price, total_price), shipping_address, subtotal, shipping_cost, total_amount_with_vat
- Multibanco block shown when `payment_status === 'pending'` and `eupago_reference` is not null
- Not-found state with back link

- `shipping_addresses` as FK relation on orders ✓

### Columns in task spec that differ from actual schema
- Task spec used `multibanco_entity`, `multibanco_reference`, `multibanco_deadline` — **actual column names are** `eupago_entity`, `eupago_reference`, `payment_deadline`
- Task spec used `shipping:shipping_addresses(...)` alias — actual FK relation name is `shipping_address:shipping_addresses(...)` (singular, matching B4 route.ts)

### Columns NOT verified in schema (flag for B8)
- `orders.paid_at` — selected but not displayed; may or may not exist; no runtime error if missing (Supabase returns null for missing columns in select)
- `orders.notes` — same; shown only if non-null
