# Landing Page Audit Checklist

## The 5-Second Test (Blink Test)

Within 5 seconds, can users answer:
- [ ] What is this page about?
- [ ] What's the main benefit/offer?
- [ ] What action should I take?
- [ ] Who is this for?

**If any answer is unclear, the page has critical issues.**

## Value Proposition

### Headline
- [ ] Headline is clear and specific (not generic)
- [ ] Headline addresses visitor's problem or desire
- [ ] Headline matches the ad/link that brought them here (message match)
- [ ] Subheadline reinforces and expands the headline

**Common Problems**: Vague headlines ("Welcome"), feature-focused instead of benefit-focused, jargon.

### Supporting Content
- [ ] Benefits are clearly explained (not just features)
- [ ] Unique value is differentiated from competitors
- [ ] Content answers "why should I care?"
- [ ] Social proof supports claims

## Call-to-Action (CTA)

### Primary CTA
- [ ] One clear primary action (focused goal)
- [ ] CTA button text is specific and action-oriented
- [ ] CTA is visually dominant (color, size, placement)
- [ ] CTA appears above the fold
- [ ] CTA is repeated at logical points down the page

**CTA Copy Examples**:
- Weak: "Submit", "Click Here", "Learn More"
- Strong: "Start Free Trial", "Get Your Report", "Book My Call"

### CTA Placement
- [ ] Primary CTA visible without scrolling
- [ ] CTA placement follows natural eye flow
- [ ] Sticky CTA for long pages (if appropriate)
- [ ] Mobile: CTA in thumb-reach zone

### Secondary Actions
- [ ] Secondary actions are visually subordinate
- [ ] No competing CTAs confusing the user
- [ ] Exit links minimized (reduce "leaks")

## Trust & Credibility

### Trust Signals
- [ ] Customer testimonials (with names, photos, specifics)
- [ ] Client logos or "as seen in" badges
- [ ] Case studies or success metrics
- [ ] Security badges (if handling sensitive data)
- [ ] Money-back guarantee or risk reversal
- [ ] Professional design quality (signals legitimacy)

### Social Proof Types
- **Testimonials**: Specific results, real names, photos
- **Numbers**: "10,000+ customers", "4.9/5 rating"
- **Logos**: Recognizable client/publication logos
- **Reviews**: Third-party ratings (G2, Trustpilot)
- **Case Studies**: Detailed success stories

### Credibility Red Flags
- Stock photos of fake customers
- Vague testimonials ("Great product!")
- No way to verify claims
- Outdated design or broken elements

## Form Optimization

### Form Design
- [ ] Minimum fields necessary (each field reduces conversion)
- [ ] Labels are clear and visible
- [ ] Required fields are marked
- [ ] Input types match expected data (email, phone, etc.)
- [ ] Error handling is helpful and specific

### Form Friction Points
- [ ] Asking for unnecessary information
- [ ] Phone number when email suffices
- [ ] Overly long forms
- [ ] CAPTCHA only when necessary
- [ ] No inline validation

**Rule of Thumb**: Each form field can reduce conversion by 5-10%

## Content Structure

### Above the Fold
- [ ] Headline + subheadline visible
- [ ] Primary CTA visible
- [ ] Relevant visual or hero image
- [ ] Clear what this page is about

### Page Flow (Typical Structure)
1. **Hero**: Headline, subheadline, CTA, hero image
2. **Problem/Pain**: Acknowledge visitor's challenge
3. **Solution**: How you solve it
4. **Benefits**: What they get (not features)
5. **Social Proof**: Testimonials, logos, case studies
6. **How It Works**: Simple process explanation
7. **FAQ**: Address objections
8. **Final CTA**: Repeat the ask

### Scanability
- [ ] Clear section headings
- [ ] Short paragraphs (3-4 lines max)
- [ ] Bullet points for lists
- [ ] Bold key phrases
- [ ] Visual breaks between sections

## Technical Performance

### Speed
- [ ] Page loads in under 3 seconds
- [ ] Images optimized (compressed, proper format)
- [ ] No render-blocking resources
- [ ] Lazy loading for below-fold content

**Impact**: 1-second delay = 7% conversion drop

### Mobile Experience
- [ ] Fully responsive layout
- [ ] Touch targets adequately sized (44x44px minimum)
- [ ] Forms usable on mobile
- [ ] No horizontal scrolling
- [ ] Text readable without zooming

## Friction Analysis

### Cognitive Friction
- [ ] Too much text overwhelming visitors
- [ ] Complex language or jargon
- [ ] Too many choices (paradox of choice)
- [ ] Unclear next steps

### Visual Friction
- [ ] Cluttered layout
- [ ] Competing visual elements
- [ ] Poor contrast or readability
- [ ] Broken visual hierarchy

### Interaction Friction
- [ ] Slow loading elements
- [ ] Broken links or buttons
- [ ] Form errors without guidance
- [ ] Unexpected behaviors
